# 工作室耗材管理系统 - 项目大纲

> 方向更新：早期设想里有“手机作为主设备，开启局域网伺服模式”。当前路线调整为：中期调研 **Docker 服务端 + PWA 渐进式**，自然语言助手提前到中期；如果 PWA 在手机扫码、离线和性能上不够稳，再保留 Android APK 作为高频扫码入口。下面内容保留为历史大纲和设计参考。

## 核心理念

> 扫码 → 算出 → 更新，10 秒内完成。软件不是产品，快速出入库流程才是。

---

## 技术选型

| 层 | 选型 | 理由 |
|---|---|---|
| 移动端 | **Capacitor APK**（HTML/JS 前端 + 原生桥接） | 一套代码出 Android APK + 后续 iOS；蓝牙、相机走原生插件，比 PWA 稳定 |
| 本地存储 | **SQL.js**（SQLite→WASM） + Capacitor Filesystem | 零服务端，App 内独立运行，离线正常扫 |
| 数据同步 | **WebDAV**（坚果云 / 自建） | 按 `updated_at` 取最新，思源笔记同款冲突模型 |
| 二维码识别 | **html5-qrcode**（Capacitor 相机插件） | 纯前端库，原生相机启动更快 |
| 蓝牙打印 | **@capacitor-community/bluetooth-le** → **精臣 BLE** | Capacitor 原生 BLE 插件，Android/iOS 双端可用 |
| 伺服模式 | **App 内嵌 HTTP Server**（Phase 4+） | 其他设备浏览器访问，数据仍在本地，同步走 WebDAV |

> **当前目标**：Android APK 一条龙——扫码 + 库存 + 打印全搞定。后续出 iOS + 伺服模式。

## 数据库设计（三张表）

### spools（耗材卷）

```sql
CREATE TABLE spools (
    id          TEXT PRIMARY KEY,          -- e.g. PLA-BLK-001
    name        TEXT NOT NULL,             -- PLA+
    brand       TEXT,
    material    TEXT,                      -- PLA / PETG / ABS / TPU
    color       TEXT,
    empty_wt    REAL NOT NULL,             -- 空卷重量(g)
    current_wt  REAL,                      -- 当前毛重(g)，NULL=未称
    location    TEXT,
    min_alarm   REAL DEFAULT 100,          -- 低于此重量(g)报警
    created_at  TEXT DEFAULT (datetime('now')),
    updated_at  TEXT DEFAULT (datetime('now'))
);
```

### parts（五金零件）

```sql
CREATE TABLE parts (
    id              TEXT PRIMARY KEY,      -- e.g. M3-INSERT
    name            TEXT NOT NULL,         -- M3热熔螺母
    unit_wt         REAL NOT NULL,         -- 单颗重量(g)
    container_wt    REAL DEFAULT 0,        -- 容器重量(g)
    estimated_qty   INTEGER,              -- 估算数量
    location        TEXT,
    min_alarm       INTEGER DEFAULT 10,   -- 低于此数量报警
    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now'))
);
```

### transactions（操作流水）

```sql
CREATE TABLE transactions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    item_type   TEXT NOT NULL,             -- 'spool' | 'part'
    item_id     TEXT NOT NULL,
    action      TEXT NOT NULL,             -- 'update' / 'adjust' / 'create'
    field       TEXT NOT NULL,             -- 'current_wt' | 'estimated_qty'
    before_val  REAL,
    after_val   REAL NOT NULL,
    source      TEXT DEFAULT 'scan',       -- 'scan' | 'manual' | 'print'
    created_at  TEXT DEFAULT (datetime('now'))
);
```

## 数据同步（WebDAV / 坚果云）

**设计原则**：所有操作写本地数据库。同步是后台静默任务，跟用户操作无耦合。服务端不负责同步——同步永远是外挂 WebDAV。

**冲突协议**（与思源笔记一致）：
- 每条记录有 `updated_at` 字段
- 同步时逐条对比，时间戳新的覆盖旧的
- 单用户场景零冲突

```
手机 App（主力）
  ├── SQL.js 本地库 ← 扫码/编辑直接写
  └── SyncService（后台定时 / 手动触发）
        ├── PUT 本地快照 → 坚果云 WebDAV
        └── GET 远程快照 → 合并
```

| 选项 | 免费容量 | 推荐 |
|---|---|---|
| 坚果云 | 1GB/月上传 | ✅ 个人直接开号 |
| 自建 WebDAV | 无限 | 有 NAS / 树莓派 |

## 操作逻辑

每次操作由按钮点击发起（设定上下文），之后扫码填充数据。

### 📦 更新库存（最多 2 次扫码）

```
点「扫库存」→ 扫重量 QR → 扫物料 QR → 自动计算 → 完成 ✅
            ↘ 或先扫物料再扫重量
            ↘ 只看一眼库存，1 次扫码就够
```

### ✨ 新建物料（最多 3 次扫码）

```
点「新建」→ 扫类似物料（克隆参数）→ 手动改差异字段 → 打印标签贴上
        → 需称重：扫重量 QR → 扫新标签 QR 确认
        → 不称重：直接扫新标签 QR 确认
```

### 🖨️ 补打标签（1 次扫码）

```
点「打印」→ 扫物料 QR → 预览 → 出纸
```

## App 页面结构

### 1. 首页
四大入口：扫库存 / 查库存 / 打标签 / 助手

### 2. 扫库存
- 状态机：等待 → 收到重量 → 收到物品 → 计算 → 展示
- 大按钮大字号，戴手套照用

### 3. 查库存
- 搜索 + 列表 + 低库存标记
- 点入可编辑参数、手调数量

### 4. 打标签
- 搜索或扫码选物料 → 预览 → 一键精臣出纸

### 5. 助手（Phase 4）
- 自然语言："黑色 PLA 还剩多少？""哪些低于报警？"

## 伺服模式（Phase 4+）

参考思源笔记的"伺服"功能——App 内嵌 HTTP Server，同局域网其他设备浏览器即可访问。

```
手机 A（主力，开了伺服）
  ├── 本地 SQL.js（数据源）
  ├── 同步 → 坚果云
  └── HTTP Server :2020
        │
        ├── 同局域网手机 B ──→ 浏览器打开 192.168.x.x:2020
        └── 台式机 C ──→ 浏览器打开查看/操作
```

- 伺服只提供 UI 入口，不存数据、不负责同步
- 所有设备共享同一份 WebDAV 数据

## 开发路线

### Phase 1 — 核心闭环（1-2天）
```
├── Capacitor 项目搭建 + Android APK 调试
├── SQL.js 本地数据库（建表 + 读写）
├── html5-qrcode 扫码
└── 两段式扫码流程（重量 → 物品）
```
**验证**：装 APK 到手机 → 扫两个 QR → 数据写入本地。

### Phase 2 — 查改完备（1 天）
```
├── 搜索 + 列表
├── 编辑（改参数、手调数量、记流水）
├── 低库存标记
└── WebDAV 同步上线
```

### Phase 3 — 打印（1-2 天）
```
├── Capacitor BLE 插件直连精臣
├── 协议无加密，组 55 55 ... aa aa 包
├── 标签模板 + 预览 + 一键打印
└── 精臣 D110 / B1 均可
```

### Phase 4 — 助手 + 伺服 + iOS
```
├── LLM 自然语言查询
├── 伺服模式（App 内嵌 HTTP Server）
├── iOS IPA 构建
└── 数据导出
```

### 后续 — ESP32 打印桥
```
├── 解决 iPhone 不能直连精臣的问题
├── mDNS 自动发现 + SoftAP 备选
└── 手机 → WiFi HTTP → ESP32 → BLE → 精臣
```

## 打印机

**精臣（Niimbot）D110 / B1**

| 型号 | 价格 | 标签宽 | 适用 |
|---|---|---|---|
| D110 | ~100 元 | 15mm | 纯文字，入门 |
| **B1** | ~150 元 | 25-50mm | QR+文字，推荐 |
| B21 | ~200 元 | 25-54mm | 更宽，热转印 |

- BLE 协议无加密，社区全逆向
- Capacitor BLE 插件直发 `55 55 ... aa aa` 包
- 全型号协议一致，代码通用
