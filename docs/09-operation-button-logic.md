# 操作和按钮逻辑

## 维护规则

这份文档记录手机 app 的操作入口、按钮含义和扫码状态机。后续每次改扫码、按钮、导航或写入流程，都要同步更新这里，避免代码逻辑和产品理解分叉。

## 当前工作区快照：0.2.0

主导航保留四个入口：

```text
扫码 / 库存 / 新增 / 流水
```

### 快捷扫码

快捷扫码是日常主入口，打开后围绕相机扫码工作。

当前按钮：

| 按钮 | 作用 |
|---|---|
| 预览扫码 | 打开页面内后置相机预览，开始识别二维码 |
| 停止 | 停止相机预览，释放摄像头 |
| 打开手电筒 / 关闭手电筒 | 相机启动后出现，设备支持 torch 时可用 |
| 原生扫码 | 打开 Android 原生扫码页；用于页面内预览扫码异常时的兜底 |
| 入库 | 当前物品满足入库条件后出现，写入库存状态 |
| 出库 | 当前物品已在库时出现，移出当前库存 |
| 恢复入库 | 当前物品已出库时出现，按新入库处理 |
| 绑定库位 | 已有待处理库位或当前物品时出现，写入库位 |
| 手动补录和测试 | 折叠入口，用于输入 payload 或点测试码 |

当前扫码反馈：

| 反馈 | 说明 |
|---|---|
| 震动 | 扫码处理成功后震动 |
| 提示音 | 扫码处理成功后播放短音 |
| 最近扫码记录 | 显示最近识别到的 payload |
| 结果卡片 | 显示查库存、更新库存或绑定库位的结果 |

当前 payload 逻辑：

| payload | 当前行为 |
|---|---|
| `msi:v1;type=spool;...` | 恢复或匹配耗材卷固定档案；默认展示同类卷数、当前重量、入库时间和出库入口 |
| `msi:v1;type=part;...` | 恢复或匹配零件固定档案；默认展示同类数量、当前估算余量和操作入口 |
| `msi:v1;type=other;...` | 恢复或匹配其他物品固定档案；默认展示状态和库位 |
| `msi:v1;type=weight;value_g=<g>` | 设置待处理重量；若已有当前物品则提示可入库或盘点 |
| `msi:v1;type=location;...` | 设置待处理库位；若已有当前物品则可直接绑定 |
| `spool:<id>` / `part:<id>` / `weight:<g>` / `location:<id>` | 旧短码兼容入口，后续标签优先使用 `msi:v1` |

当前上下文不再靠固定模式按钮驱动，而是靠扫码内容驱动：

```text
currentProfile   当前固定档案
currentState     当前库存状态，可为空
pendingWeight    待处理重量
pendingLocation  待处理库位
```

扫物品码默认就是查库存；扫完整 `msi:v1` 标签但本地没有状态时，只建档，不默认算入库。

### 查库存

查库存页只负责查看和筛选，不负责新增。

当前按钮和控件：

| 控件 | 作用 |
|---|---|
| 搜索框 | 搜索 ID、名称、材料、颜色、库位 |
| 类型筛选 | 全部 / 耗材卷 / 零件 / 其他 |
| 状态筛选 | 在库 / 已出库 / 仅档案 |
| 低库存 | 只看低库存物品 |

数据入口：

| 控件 | 作用 |
|---|---|
| 本地自动保存 | 默认数据落点，扫码和编辑后立即写本地 |
| 云同步未接入 | 状态提示，后续接 WebDAV 同步 |
| 备份和调试 | 折叠入口，不属于日常库存主流程 |
| 合并导入快照 | 临时调试/恢复，用快照合并本地数据 |
| 覆盖恢复快照 | 灾难恢复，用快照替换本地数据 |
| 导出本地备份 | 导出完整快照用于备份和调试 |

点击库存行会跳到“新增”页编辑该物品。

`导出当前列表` 不再作为正式按钮保留。我们规划的是本地一份主数据、云端一份同步副本；筛选列表只是查看口径，不承担同步或交接。

### 新增

新增页负责物品资料维护和标签生成。

当前按钮：

| 按钮 | 作用 |
|---|---|
| 新建 | 清空表单，准备创建物品 |
| 克隆 | 从当前物品复制字段，生成一个新物品草稿 |
| 归档 / 恢复 | 停用或恢复当前物品 |
| 生成 ID | 按类型和字段生成可用 ID |
| 保存物品 | 新增或更新固定档案 |
| 生成标签 | 为已存在物品生成 `msi:v1` 二维码 |

当前约定：

```text
新增物品 -> 保存 -> 自动选中该物品 -> 可生成标签
```

### 流水

流水页当前只展示改动记录。

当前记录来源：

| 操作 | 写入流水 |
|---|---|
| 入库 | 是 |
| 盘点称重更新耗材卷重量 | 是 |
| 盘点称重更新零件估算数量 | 是 |
| 绑定库位 | 是 |
| 出库 | 是 |
| 恢复入库 | 是 |

## 当前精简结果

扫码已经从“模式驱动”改成“上下文驱动”。用户不需要先选择查库存、盘点称重或绑定库位，扫码内容和当前上下文决定下一步。

当前状态：

```text
activeItem     当前物品
pendingWeight  待处理重量
pendingLocation 待处理库位
```

当前操作逻辑：

| 扫码顺序 | 行为 |
|---|---|
| 扫物品码 | 查库存，并设为当前物品 |
| 扫重量码 -> 扫物品码 | 满足条件时显示入库或盘点写入入口 |
| 扫物品码 -> 扫重量码 | 满足条件时显示入库或盘点写入入口 |
| 扫库位码 -> 扫物品码 | 显示绑定库位入口 |
| 扫物品码 -> 扫库位码 | 显示绑定库位入口 |
| 扫已在库物品码 | 显示出库入口 |
| 扫已出库物品码 | 显示恢复入库入口，按新入库处理 |

扫码页按钮保留为：

```text
预览扫码 / 停止 / 手电筒 / 原生扫码 / 当前可执行动作 / 手动补录
```

当前没有固定“查库存”按钮；扫物品码就是查库存。固定模式按钮不再作为主流程入口。

## 0.2 目标逻辑

0.2 以 [12-msi-v1-and-0.2-scope.md](./12-msi-v1-and-0.2-scope.md) 为准。

核心规则：

| 规则 | 说明 |
|---|---|
| `msi:v1` 是长期推荐二维码协议 | 旧短码继续兼容 |
| 二维码只存固定档案 | 当前重量、数量、库位、状态都不进二维码 |
| 扫未知完整标签只建档 | 只创建 `ItemProfile`，不创建 `ItemState`，不计入库存 |
| 入库是明确动作 | 满足各类型条件后写 `stock_in` 流水 |
| 库位可选 | 所有类型都可以无库位入库，后续再扫码绑定 |
| `other` 简单登记 | 支持状态和库位，不做余量计算 |
| 业务日期按天 | `created_on / stocked_on / checked_out_on` 使用 `YYMMDD` |
| 流水保留精确时间 | `created_at` 必须带时区，并记录 `device_id` |

入库按钮出现条件：

| 类型 | 入库按钮条件 |
|---|---|
| `spool` | 已识别档案，并已有当前重量 |
| `part` | 已识别档案，并已有数量或总重量 |
| `other` | 已识别档案，用户确认即可 |

扫码恢复本地数据时：

```text
扫到完整 msi:v1 标签
  -> 恢复固定档案
  -> 显示“已建档，未入库”
  -> 等待用户称重 / 输入数量 / 确认入库
```

## 风险点

后续完善时要特别注意：

- 重复扫码防抖，避免连续写两次。
- 写入后要有明确结果、震动和声音。
- 写错后的“撤销上一步”比每次弹确认更高效。
- 扫到未知物品时不能吞掉待处理重量或库位。
- 扫到新重量时应覆盖旧的待处理重量。
- 扫到新库位时应覆盖旧的待处理库位。
- Android 原生扫码只应该把 payload 交给核心，不直接改库存。
