# 库存筛选

## 为什么单独拆 `filters.js`

库存列表筛选不是页面渲染细节，也不是库存计算本身。

它的职责是：

```text
物品列表 + 筛选条件 -> 应该显示哪些物品
```

单独拆出来的好处：

- 页面只负责读取输入框、复选框和渲染。
- 后续低库存报表、助手查询、同步前检查都能复用筛选规则。
- 筛选可以用 Node 测试，不依赖浏览器。

## 当前支持

- 搜索：ID、名称、品牌、材料、颜色、库位。
- 类型：全部 / 耗材卷 / 零件。
- 归档状态：启用 / 全部状态 / 已归档。
- 低库存：只显示低于报警值的物品。
- 导出当前筛选结果：只导出当前列表命中的物品和相关流水。

低库存判断来自 `core/inventory.js` 的 `getStockInfo()`，所以列表、扫码查询、后续报表使用同一套库存口径。

默认归档口径：

- 库存列表默认只看启用物品。
- 低库存总数不统计已归档物品。
- 导出当前列表遵守页面当前筛选条件。
